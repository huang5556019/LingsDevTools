# 技术方案：历史记录管理 (History)

## 1. 功能概述
历史记录管理功能用于统一管理各类工具的使用历史，包括 HTTP 请求历史、常用正则表达式收藏、以及各类转换工具的历史记录。用户可以查看、重用、管理历史记录，提高工具使用效率。

## 2. 技术架构

### 2.1 架构概述
- **主进程**：负责数据库操作和 IPC 通信处理
- **渲染进程**：负责前端 UI 展示和用户交互
- **数据流向**：工具操作 → 渲染进程 → IPC 通信 → 主进程 → 数据库存储/读取 → IPC 通信 → 渲染进程 → UI 展示

### 2.2 技术栈
- **前端**：React 18.x + TypeScript + Tailwind CSS + Zustand
- **数据库**：SQLite (better-sqlite3)
- **IPC 通信**：Electron 原生 IPC
- **状态管理**：Zustand

## 3. 数据模型设计

### 3.1 现有表结构
```sql
-- 历史记录表
CREATE TABLE IF NOT EXISTS history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  tool_type TEXT NOT NULL,  -- 工具类型（http、regex、base64、url、json、timestamp）
  input TEXT NOT NULL,      -- 输入数据
  output TEXT NOT NULL,     -- 输出数据
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP  -- 创建时间
);

-- 收藏表
CREATE TABLE IF NOT EXISTS favorites (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  tool_type TEXT NOT NULL,  -- 工具类型
  name TEXT NOT NULL,       -- 收藏名称
  data TEXT NOT NULL,       -- 收藏数据
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP  -- 创建时间
);
```

### 3.2 数据模型扩展
- **history 表**：已满足需求，无需修改 → AC-012
- **favorites 表**：用于收藏常用的正则表达式等 → AC-002

## 4. API 接口设计

### 4.1 IPC 接口

#### 4.1.1 历史记录相关
- **history:save**
  - 功能：保存历史记录
  - 参数：`{ tool_type: string, input: string, output: string }`
  - 返回：`{ success: boolean, id?: number, error?: string }`
  - 对应 AC：AC-001, AC-002, AC-003, AC-011, AC-012

- **history:getAll**
  - 功能：获取所有历史记录
  - 参数：`{ limit?: number, offset?: number }`
  - 返回：`{ success: boolean, data: History[], error?: string }`
  - 对应 AC：AC-004, AC-013

- **history:delete**
  - 功能：删除历史记录
  - 参数：`{ ids: number[] }`
  - 返回：`{ success: boolean, error?: string }`
  - 对应 AC：AC-006, AC-014

- **history:clear**
  - 功能：清空历史记录
  - 参数：`{}`
  - 返回：`{ success: boolean, error?: string }`
  - 对应 AC：AC-014

#### 4.1.2 收藏相关
- **favorites:save**
  - 功能：保存收藏
  - 参数：`{ tool_type: string, name: string, data: string }`
  - 返回：`{ success: boolean, id?: number, error?: string }`
  - 对应 AC：AC-002

- **favorites:getAll**
  - 功能：获取所有收藏
  - 参数：`{ tool_type?: string }`
  - 返回：`{ success: boolean, data: Favorite[], error?: string }`
  - 对应 AC：AC-002

- **favorites:delete**
  - 功能：删除收藏
  - 参数：`{ id: number }`
  - 返回：`{ success: boolean, error?: string }`
  - 对应 AC：AC-002

## 5. 核心逻辑实现

### 5.1 数据库操作

#### 5.1.1 历史记录操作
- **保存历史记录**
  1. 接收 IPC 调用，获取参数
  2. 检查历史记录数量，若达到上限（如 1000 条），删除最旧的记录 → AC-008
  3. 插入新记录到 history 表
  4. 返回操作结果

- **获取历史记录**
  1. 接收 IPC 调用，获取参数
  2. 执行查询，按 created_at 倒序排列 → AC-013
  3. 应用分页参数（如果有）
  4. 返回查询结果

- **删除历史记录**
  1. 接收 IPC 调用，获取要删除的 ID 列表
  2. 执行删除操作 → AC-014
  3. 返回操作结果

### 5.2 IPC 通信实现
- 在 `src/main/ipc/handlers.ts` 中注册历史记录相关的 IPC 处理器
- 实现错误处理，确保数据库操作失败时能返回错误信息 → AC-007

### 5.3 前端状态管理
- 使用 Zustand 创建 historyStore
- 管理历史记录的加载状态、数据状态和错误状态
- 提供添加、删除、获取历史记录的方法

## 6. 前端组件设计

### 6.1 组件结构
- **HistoryList 组件**
  - 展示历史记录列表
  - 支持按工具类型过滤
  - 支持选择和批量删除
  - 支持点击历史记录项加载数据到对应工具 → AC-005

- **HistoryItem 组件**
  - 展示单条历史记录
  - 显示工具类型、操作时间、输入/输出摘要
  - 支持点击加载数据

- **FavoriteList 组件**
  - 展示收藏列表
  - 支持按工具类型过滤
  - 支持删除收藏
  - 支持点击加载收藏数据

### 6.2 布局设计
- 在左侧边栏添加历史记录入口
- 历史记录页面采用列表视图，支持分页
- 每条历史记录显示关键信息，鼠标悬停显示完整信息

## 7. 异常处理

### 7.1 数据库异常
- **连接失败**：显示保存失败提示，不影响工具正常使用 → AC-007
- **查询失败**：显示加载失败提示，提供重试按钮
- **插入失败**：显示保存失败提示，记录错误日志

### 7.2 数据异常
- **数据格式异常**：显示数据格式错误提示，不影响其他历史记录的加载 → AC-010
- **已删除记录**：显示记录不存在提示 → AC-009

### 7.3 IPC 通信异常
- 实现超时处理
- 提供错误重试机制
- 记录详细的错误日志

## 8. 性能优化

### 8.1 数据库优化
- 为 history 表的 created_at 字段创建索引，加速排序查询
- 实现历史记录数量限制，防止数据库过度增长
- 使用批量操作减少数据库 I/O 次数

### 8.2 前端优化
- 实现虚拟列表，支持大量历史记录的高效渲染
- 使用防抖处理，减少频繁的数据库操作
- 缓存历史记录数据，减少重复查询

### 8.3 网络优化
- 批量处理 IPC 通信，减少通信次数
- 压缩传输数据，减少通信量
