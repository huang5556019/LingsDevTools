# 技术栈选型

## 1. 核心决策 (Key Decisions)
*简述为什么选择这套技术栈？*
> 选择 Electron + React + TypeScript + SQLite 技术栈，因为：
> - Electron 已被确定为桌面应用框架，成熟稳定，一次开发双平台运行
> - React 生态丰富，组件化开发适合构建工具类应用
> - TypeScript 提供类型安全，减少开发错误，提升代码可维护性
> - SQLite 轻量级本地数据库，无需额外服务，适合存储历史记录
> - 整体技术栈成熟稳定，社区资源丰富，开发效率高

## 2. 前端 (Frontend)
*   **框架**: React 18.x
    *   原因：组件化开发，生态丰富，大量成熟的 UI 组件库可选
*   **样式方案**: Tailwind CSS
    *   原因：原子化 CSS，快速构建传统工具风格 UI，无需编写大量自定义 CSS
*   **状态管理**: Zustand
    *   原因：轻量级状态管理，API 简洁，适合工具类应用的状态需求
*   **UI 组件库**: Ant Design 或 Mantine
    *   原因：成熟的组件库，提供传统工具风格的 UI 组件

## 3. 桌面应用框架 (Desktop Framework)
*   **核心框架**: Electron 28.x
    *   原因：已确定技术栈，跨平台支持好，社区成熟
*   **构建工具**: Vite + electron-vite
    *   原因：快速构建，开发体验好，支持热重载
*   **IPC 通信**: Electron 原生 IPC
    *   原因：直接使用 Electron 提供的主进程与渲染进程通信机制

## 4. 数据存储 (Database)
*   **本地数据库**: SQLite (通过 better-sqlite3)
    *   原因：轻量级，无需额外数据库服务，适合本地存储历史记录
    *   用途：存储 HTTP 请求历史、正则表达式收藏、各类工具的使用历史
*   **配置存储**: Electron 的 app.getPath('userData') + JSON 文件
    *   原因：简单的配置数据直接存储在 JSON 文件中

## 5. 工具库 & 依赖
*   **HTTP 请求**: axios
    *   原因：功能强大，支持各种 HTTP 方法和配置
*   **WebSocket**: 原生 WebSocket API
    *   原因：浏览器原生支持，无需额外依赖
*   **时间处理**: day.js
    *   原因：轻量级，API 友好，适合时间戳转换
*   **JSON 格式化**: json-format 或自定义实现
    *   原因：简单的格式化需求，可自定义实现或使用轻量级库
*   **编码解码**: 浏览器原生 API (btoa/atob, encodeURIComponent 等)
    *   原因：浏览器原生支持，无需额外依赖

## 6. 开发工具 (Dev Tools)
*   **包管理器**: pnpm
    *   原因：快速、节省磁盘空间，比 npm/yarn 更高效
*   **代码格式化**: Prettier
    *   原因：统一代码风格
*   **代码检查**: ESLint
    *   原因：检测代码问题，提升代码质量
*   **类型检查**: TypeScript Compiler
    *   原因：编译时类型检查

## 7. 构建 & 打包
*   **构建工具**: Vite + electron-vite
    *   原因：快速构建，开发体验好
*   **打包工具**: electron-builder
    *   原因：成熟的 Electron 打包工具，支持 Windows 和 macOS
*   **输出格式**:
    *   Windows: .exe 安装包
    *   macOS: .dmg 安装包
