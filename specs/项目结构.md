# 项目结构

## 1. 目录树 (Directory Tree)
```
DevTools/
├── specs/                  # 项目核心定义 (产品/技术/结构/规范)
├── docs/                   # 项目文档
├── src/                    # 源代码
│   ├── main/               # Electron 主进程
│   │   ├── ipc/            # IPC 通信处理
│   │   ├── database/       # 数据库操作
│   │   └── index.ts        # 主进程入口
│   ├── preload/            # 预加载脚本
│   │   └── index.ts        # 预加载脚本入口
│   └── renderer/           # 渲染进程 (React 前端)
│       ├── components/     # 通用组件
│       ├── modules/        # 业务模块
│       │   ├── encoder/    # 编码解码工具包
│       │   ├── network/    # 网络工具包
│       │   ├── data/       # 数据处理工具包
│       │   └── history/    # 历史记录管理
│       ├── store/          # 状态管理 (Zustand)
│       ├── styles/         # 样式文件
│       ├── App.tsx         # React 应用入口
│       └── main.tsx        # 渲染进程入口
├── resources/              # 静态资源
│   ├── icons/              # 应用图标
│   └── images/             # 图片资源
├── tests/                  # 测试代码
│   ├── unit/               # 单元测试
│   └── e2e/                # 端到端测试
├── .gitignore
├── .env.example
├── .prettierrc
├── .eslintrc.js
├── tsconfig.json
├── vite.config.ts
├── electron-builder.yml
├── package.json
└── README.md
```

## 2. 关键文件说明 (Key Files)
*   **package.json**: 项目依赖管理和脚本配置。
*   **vite.config.ts**: Vite 构建配置。
*   **electron-builder.yml**: Electron 打包配置。
*   **tsconfig.json**: TypeScript 配置。
*   **.eslintrc.js**: ESLint 代码检查配置。
*   **.prettierrc**: Prettier 代码格式化配置。
*   **.env.example**: 环境变量示例（禁止上传敏感信息）。
*   **README.md**: 项目入口文档，必须包含"如何启动"说明。

## 3. 模块说明 (Module Description)

### 主进程 (src/main/)
*   **src/main/index.ts**: Electron 主进程入口，创建窗口、管理应用生命周期。
*   **src/main/ipc/**: IPC 通信处理，处理来自渲染进程的请求。
    *   `ipc-database.ts`: 数据库相关 IPC 处理。
    *   `ipc-network.ts`: 网络请求相关 IPC 处理。
*   **src/main/database/**: SQLite 数据库操作。
    *   `schema.ts`: 数据库表结构定义。
    *   `history.ts`: 历史记录数据操作。
    *   `init.ts`: 数据库初始化。

### 预加载脚本 (src/preload/)
*   **src/preload/index.ts**: 预加载脚本，暴露安全的 API 给渲染进程。

### 渲染进程 (src/renderer/)
*   **src/renderer/modules/encoder/**: 编码解码工具包。
    *   `Base64Tool.tsx`: Base64 编码/解码组件。
    *   `UrlTool.tsx`: URL 编码/解码组件。
    *   `JsonTool.tsx`: JSON 格式化/压缩组件。
*   **src/renderer/modules/network/**: 网络工具包。
    *   `HttpTool.tsx`: HTTP 请求测试组件。
    *   `WebSocketTool.tsx`: WebSocket 调试组件。
*   **src/renderer/modules/data/**: 数据处理工具包。
    *   `TimestampTool.tsx`: 时间戳转换组件。
    *   `RegexTool.tsx`: 正则表达式测试组件。
*   **src/renderer/modules/history/**: 历史记录管理。
    *   `HistoryList.tsx`: 历史记录列表组件。
    *   `useHistory.ts`: 历史记录 Hook。
*   **src/renderer/components/**: 通用组件。
    *   `Layout.tsx`: 整体布局组件。
    *   `Sidebar.tsx`: 侧边栏导航。
    *   `ToolContainer.tsx`: 工具容器组件。
*   **src/renderer/store/**: Zustand 状态管理。
    *   `useAppStore.ts`: 应用全局状态。
    *   `useHistoryStore.ts`: 历史记录状态。
*   **src/renderer/styles/**: 样式文件。
    *   `global.css`: 全局样式。
    *   `tailwind.css`: Tailwind CSS 配置。
*   **src/renderer/App.tsx**: React 应用根组件。
*   **src/renderer/main.tsx**: 渲染进程入口文件。

## 4. 文件放置规则 (Placement Rules)
*   **工具组件**：放在 `src/renderer/modules/[对应模块]/`，每个工具一个独立文件。
*   **业务逻辑**：工具的核心逻辑放在对应模块目录下，禁止直接写在 UI 组件中。
*   **通用组件**：放在 `src/renderer/components/`。
*   **状态管理**：放在 `src/renderer/store/`。
*   **IPC 处理**：放在 `src/main/ipc/`。
*   **数据库操作**：放在 `src/main/database/`。
*   **工具函数**：放在对应模块的 `utils/` 子目录或 `src/renderer/shared/utils/`。

## 5. 扩展性设计
*   新增工具时，在 `src/renderer/modules/` 下创建对应目录和组件。
*   在侧边栏配置中添加新工具的导航项。
*   如需历史记录支持，在数据库 schema 中添加对应表结构。
