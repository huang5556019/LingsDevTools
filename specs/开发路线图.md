# 项目开发路线图 (Development Roadmap)

## 0. 策略概览
*   **开发策略**: 优先搭建 Electron 基础框架和数据库基础设施，然后开发核心工具模块，最后实现历史记录管理功能
*   **MVP 目标**: 实现编码解码、数据处理和网络工具的核心功能，具备基本的历史记录能力

## 1. 里程碑一：MVP (核心闭环)
> 目标：实现最基础的工具功能，完成从 0 到 1 的闭环。

- [ ] **Phase 1.1: 基础设施搭建 (Infrastructure)**
    *   **模块**: `Electron 基础框架` / `数据库初始化` / `通用组件`
    *   **说明**: 搭建 Electron 项目框架，配置 TypeScript、Vite 和 Tailwind CSS，初始化 SQLite 数据库，创建通用布局组件
    *   **具体任务**:
        - 初始化 Electron + React + TypeScript 项目
        - 配置 Vite 和 electron-vite
        - 配置 Tailwind CSS
        - 实现主进程入口和窗口创建
        - 实现预加载脚本和 IPC 通信基础
        - 初始化 SQLite 数据库和表结构
        - 创建通用布局组件 (Layout, Sidebar, ToolContainer)

- [ ] **Phase 1.2: 编码解码工具包 (Encoder)**
    *   **模块**: `encoder`
    *   **说明**: 实现 Base64 编码/解码、URL 编码/解码、JSON 格式化/压缩功能
    *   **具体任务**:
        - 实现 Base64Tool 组件
        - 实现 UrlTool 组件
        - 实现 JsonTool 组件

- [ ] **Phase 1.3: 数据处理工具包 (Data)**
    *   **模块**: `data`
    *   **说明**: 实现时间戳转换和正则表达式测试功能
    *   **具体任务**:
        - 实现 TimestampTool 组件
        - 实现 RegexTool 组件

- [ ] **Phase 1.4: 网络工具包 (Network)**
    *   **模块**: `network`
    *   **说明**: 实现 HTTP 请求测试和 WebSocket 调试功能
    *   **具体任务**:
        - 实现 HttpTool 组件
        - 实现 WebSocketTool 组件

- [ ] **Phase 1.5: 历史记录管理 (History)**
    *   **模块**: `history` / `database`
    *   **说明**: 实现历史记录的保存、读取和管理功能
    *   **具体任务**:
        - 实现数据库历史记录表结构
        - 实现历史记录 IPC 通信
        - 实现 HistoryList 组件
        - 实现 useHistory Hook

## 2. 里程碑二：功能完善 (Feature Complete)
> 目标：完善现有功能，提升用户体验。

- [ ] **Phase 2.1: 功能优化与测试**
    *   **模块**: 所有工具模块
    *   **说明**: 优化工具功能，添加错误处理，编写测试
    *   **具体任务**:
        - 优化各工具的用户界面
        - 添加完整的错误处理
        - 编写单元测试
        - 进行端到端测试

- [ ] **Phase 2.2: 打包与发布**
    *   **模块**: `构建配置`
    *   **说明**: 配置 electron-builder，构建 Windows 和 macOS 安装包
    *   **具体任务**:
        - 配置 electron-builder
        - 构建 Windows .exe 安装包
        - 构建 macOS .dmg 安装包

## 3. 里程碑三：运营与优化 (Ops & Optimization)
> 目标：性能优化、文档完善和后续扩展。

- [ ] **Phase 3.1: 性能优化**
    *   **模块**: 所有模块
    *   **说明**: 优化应用启动速度和工具响应速度
    *   **具体任务**:
        - 优化应用启动时间
        - 优化工具操作响应速度
        - 优化数据库查询性能

- [ ] **Phase 3.2: 文档与部署**
    *   **模块**: `文档`
    *   **说明**: 完善项目文档，准备部署
    *   **具体任务**:
        - 完善 README.md
        - 编写用户使用文档
        - 准备发布说明

---
**使用指南**:
1.  **功能开发**: 按照顺序，依次开发每个功能模块。
2.  每完成一个模块，请回来勾选进度。
3.  **开发提示**:
    - 基础设施搭建是后续所有功能的基础，请优先完成
    - 编码解码和数据处理工具相对独立，可并行开发
    - 网络工具需要依赖 IPC 通信，建议在基础设施完成后开发
    - 历史记录管理依赖数据库和各工具模块，建议最后开发
